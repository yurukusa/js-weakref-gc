Strong vs Weak References

A strong reference (normal variable assignment) prevents garbage collection. The object stays alive as long as any strong reference exists.

A weak reference (WeakRef, WeakMap, WeakSet) does NOT prevent garbage collection. If the only remaining references to an object are weak, the GC can reclaim it at any time.

WeakMap

  • Keys must be objects (not primitives)
  • Keys are held weakly — if the key object is GC'd, the entry is automatically removed
  • Not iterable (you can't list all entries)
  • Use case: Associating metadata with objects without preventing their GC

WeakSet

  • Values must be objects
  • Values are held weakly — GC can reclaim them
  • Not iterable
  • Use case: Tracking which objects have been "seen" or "processed"

WeakRef

  • Wraps an object with a weak reference
  • ref.deref() returns the object if still alive, or undefined if GC'd
  • Use case: Caches where you want to keep objects only while they're used elsewhere

FinalizationRegistry

  • Registers a callback that fires when an object is garbage collected
  • Callback receives a "held value" you specify at registration
  • Use case: Cleanup (closing files, releasing resources) when objects are GC'd
  • Warning: Timing is not guaranteed — callback may fire much later or never
